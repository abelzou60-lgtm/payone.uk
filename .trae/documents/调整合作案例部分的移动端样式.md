# 调整合作案例部分的移动端样式

## 问题分析
当前合作案例部分在移动端的实现存在以下问题：
1. `.cases-track` 使用了 `flex` 布局但未设置 `flex-wrap: wrap`，导致卡片无法换行
2. 移动端设置了 `overflow-x: hidden` 但卡片仍在单行排列，可能导致显示不完整
3. 需要确保在移动端固定显示两列卡片布局

## 解决方案

### 修改 CSS 样式
1. **在移动端媒体查询中添加 `flex-wrap: wrap`**
   - 为 `.cases-track` 添加 `flex-wrap: wrap` 属性，使卡片能够自动换行
   - 保持 `.case-card` 的 `flex: 0 0 calc(50% - 10px)` 设置，确保每张卡片占满半行

2. **调整容器宽度**
   - 确保 `.cases-slider` 在移动端有正确的宽度设置，允许卡片正常换行

3. **移除不必要的滚动行为**
   - 保持 `overflow-x: hidden` 以防止水平滚动条出现
   - 调整 `scrollCases` 函数，使其在移动端不执行滚动操作

### 具体修改点
1. **修改 CSS 中的移动端媒体查询**（大约在第 754 行附近）
   - 为 `.cases-track` 添加 `flex-wrap: wrap`

2. **调整 JavaScript 滚动函数**（大约在第 992 行附近）
   - 修改 `scrollCases` 函数，在移动端屏幕尺寸下不执行滚动

## 预期效果
- 在移动端设备上，合作案例部分将以两列网格布局显示
- 每行固定显示两张卡片，自动换行
- 保持与现有设计风格一致，包括卡片间距、大小和样式
- 移除不必要的水平滚动功能，提供更清晰的网格布局
